是否赞同写法 4

写法四：“取决于”

通过题目的最后一句话 “To what extent do you agree or disagree?“ 我们可以确定题型为是否赞同，在什么程度上你同意或不同意。
注意：是否题型是你自己的观点，不需要别人怎么认为。

题目中的观点可以一分为二地来看待，但可以取决于不同的条件。

文体结构

1. Introduction: 改写题目，给出个人观点预告 —— 取决于
2. Body Paragraph 1: 条件情况一
3. Body Paragraph 2: 条件情况二
4. Conclusion: 总结，重复个人观点

动态规划问题

斐波那契数列

递归写法

int fib(int N) {
   if (N == 1 || N == 2) return 1;
   return fib(N - 1) + fib(N - 2);
}

递归算法的时间复杂度：子问题个数乘以解决一个子问题需要的时间
子问题个数： O(2^n)
解决一个子问题的时间：在本算法中没有循环，只有f(n-1)+f(n-2)这一个加法操作，所以时间复杂度为： O(1)
所以该算法的时间复杂度为： O(2^n)
通过递归树可以发现有很多的重复计算，如：f(18)。

这就是动态规划问题的第一个性质：重叠子问题。

双边讨论写法 1

写法一：“Both 两者都”

通过题目的最后一句话 “Discuss both views and give you own opinion.“ 我们可以确定题型为Discuss，讨论这两个观点并给出你自己的观点。
注意：Discuss题型要求平衡地论证题目给出的两个观点，核心段字数要大致相同。

当我们遇到题目给出的两个观点都想同意时，个人观点就可以为两者都同意。因此称这种写法为“Both 两者都”。

文体结构

1. Introduction: 改写题目，给出个人观点预告
2. Body Paragraph 1: 观点1
3. Body Paragraph 2: 观点2
4. Conclusion: 总结，个人观点：两者缺一不可
